Render Markup

Render HTML markup below your the code sample.

How to use

You don't need to do anything; Prism highlighted codes blocks with language-markup class will have the HTML rendered beneath the highlighted markup.

This plugin is only applicable for the markup grammer (most likely when highlighting HTML). Prism doesn't distinguish between XML and HTML, so while this plugin will insert XML into your page it's probably not going to be overly useful (although see the SVG example below where you may want this).

Disabling Render Markup per code block

If you wish to disable this plugin for a specific code block, specify data-render-markup="off" on the <pre>:

<pre class="language-markup" data-render-markup="off">

Disabling Render Markup globally

If you wish to disable this plugin globally by default, specify data-render-markup="off" on the <script> that included Prism to your page. You can then specify data-render-markup="on" for just the <pre> tags of those code blocks you wish to enable this plugin.

Styling

To assist with custom styling, the rendered markup is wrapped in a <div> with a prism-markup-render class, and the highlighted code block will have a prism-markup-rendered class (note: render vs rendered).

Examples

Basic markup

<section>
	<h2>Heading</h2>
	<p><a href="/download.html">Download</a> <em>Prism</em> now.</p>
	<p class="foo" onclick="alert('Foo');">CSS and JavaScript works too</p>
	<style>
		.foo { cursor: pointer; text-transform: uppercase; }
	</style>
</section>

Specify no render

<section>
	No markup rendered here, because of the <pre data-render-markup="off"> attribute.
</section>

Inline code

This is some <i>inline</i> code, naturally with no rendering.

SVG

This plugin will work with the File Highlight plugin, and can render non-HTML.
